# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

## Слой Model
Объединяет всю логику по хранению и правилам работы с данными, изолируя их от остальных слоев. Включает следующие классы:
-Products - хранит каталог товаров, предоставляет методы для установки списка товаров, их получения и поиска товара по id;
-Basket - хранит товары, добавленные в корзину, и предоставляет методы для добавления, удаления, очистки корзины, подсчёта суммы и количества товаров;
-Buyer - хранит данные покупателя: адрес, телефон, email, способ оплаты, а также выполняет их валидацию;
-ApiService - отвечает за получение каталога товаров и отправку заказа на сервер;

## Слой Presenter
Представлен файлом main.ts. Он управляет логикой и поведением приложения: создает экземпляры всех моделей, вызывает методы, выполняет тесты методов, запросы к серверу и выводит необходимые данные в консоль. 

## Слой View
На данном этапе работы над проектом не реализован


### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

#### Данные

##### тип BuyerErrors = Partial<Record<keyof IBuyer, string>>:
Хранит единый тип для валидации ошибки при обработке данных, вводимых пользователем. 

##### тип export type Tpayment = "card" | "cash" | "";
Хранит единый тип с доступными вариантами оплаты. 

##### Интерфейс Iproduct:
Регламентирует формат данных о продукте, получаемых с сервера и приводит их к стандартизированному формату. Содержит следующие поля:
-id: хранит id продукта и позволяет находить нужный в списке товаров;
-title: хранит название продукта;
-description: хранит подробное описание товара;
-image: хранит ссылку на изображение товара;
-category: хранит данные о категории товара.

Перечисленные выше поля принимают формат строки.

-price: хранит информацию о цене. Может быть в формате числа или принимать значение null, если цена для товара не назначена. 

##### Интерфейс IBuyer:
Регламентирует формат данных о покупателе и приводит их к стандартизированному формату. Интерфейс описывает данные покупателя на стороне клиента и отправляет их на сервер при оформлении заказа. Содержит следующие поля:
-payment: хранит данные о выбранном способе оплаты, принимает строковое значение card - при выборе карты, как способа оплаты, cash - при выборе наличных. Также может принимать пустую строку в случае, когда способ оплаты не выбран;
address: в формате строки хранит данные об адресе, указанном пользователем в момент оформления заказа;
email: в формате строки хранит email покупателя;
phone: в формате строки хранит номер телефона покупателя.

##### Интерфейс IProductsResponse
Необходим для типизации полного ответа с сервера, который приходит в виде объекта. Содержит два поля:
-total: показывает общее количество товаров в каталоге на сервере;
-items: хранит массив товаров, который присылает сервер.

##### Интерфейс IOrderResponse
Регламентирует формат ответа сервера после успешного заказа. Содержит два поля: 
-id: уникальный id заказа;
-total: подтвержденная сумма заказа. 

##### Интерфейс IOrderRequest
Регламентирует формат данных, отправляемых на сервер после совершения заказа. Содержит следующие поля:
-payment: выбранный пользователем способ оплаты;
-email:  эл. почта пользователя в формате строки;
-phone: номер телефона пользователя в формате строки;
-address: адрес, указанный покупателем, в формате строки;
-total: содержит итоговую сумму заказа в формате числа;
-items: массив из строк, который хранит id выбранных товаров.


###### Модели данных:

###### Класс Products:
Отвечает за хранение и предоставление данных о товарах. 

Поля класса:
`private items: IProduct[] = []` - пустой массив, использующийся для хранения полученных с сервера товаров. Формат элементов массива определяет интерфейс Iproduct. 
`private selectedProduct: IProduct | null = null;` - свойство, которое хранит выбранный товар в формате интерфейса IProduct, если ничего не выбранно, принимает значение null. 

Методы класса:
`public setItems(items: IProduct[]): void { this.items = items;}` - метод принимает массив товаров Iproduct, полученный с сервера, и кладет его в хранилище items. Метод ничего не возврашает. 
`public getItems(): IProduct[] { return this.items; }` - возврашает полученный и сохраненный в приватном поле items список товаров. Дает безопасный доступ к каталогу из других частей приложения, не позволяя менять данные напрямую.
`public getProductById(id: string): IProduct | undefined { return this.items.find((item) => item.id === id); }` - ищет товар в каталоге по id. Принимает id в виде строки из ранее сохраненного массива в items, возвращает объект в формате Iproduct, если товар найден, и undefined - если искомого товара нет.
`public setSelectedProduct(product: IProduct): void { this.selectedProduct = product; }` - метод для "установки" сохранения выбранного товара в приватное поле selectedProduct. Ничего не возвращает. 
`public getSelectedProduct(): IProduct | null { return this.selectedProduct; }` - метод возвращает текущее значение приватного поля selectedProduct, то есть - выбранного товара. Возвращает либо объект в формате Iproduct, либо null, если товар не выбран. 
`public clearSelectedProduct(): void { this.selectedProduct = null; }` - метод сбрасывает состояние поля selectedProduct до "ничего не выбрано". Ничего не возвращает, просто обнуляет состояние поля. 


###### Класс Basket
Отвечает за работу с товарами в корзине: доступ к списку всех добавленных в корзину товаров, добавление, удаление, очистка.

Поля класса:
`private items: IProduct[] = []` - пустой массив, использующийся для хранения товаров, добавленных в корзину. Формат элементов массива определяет интерфейс Iproduct.

Методы класса:
`public getItems(): IProduct[] { return this.items; }` - метод возвращает текущий список товаров из корзины. 
`public addItem(product: IProduct): void { this.items.push(product); }` - метод добавляет выбранный товар в корзину. На входе получает готовый объект товара соответствующий интерфейсу IProduct и кладет его в массив, который хранит товары корзины. Ничего не возвращает. 
`public removeItem(id: string): void { this.items = this.items.filter(item => item.id !== id); }` - метод удаляет выбранный товар из корзины по id. С помощью filter создается новый массив, в котором есть все товары, кроме выбранного на удаление. 
`public clear(): void { this.items = []; }` - метод полностью очищает корзину, заменяя массив товаров пустым. 
`public getTotal(): number { return this.items.reduce((sum, item) => sum + (item.price ?? 0), 0); }` - метод считает стоимость всех товаров, добавленных в корзину. 
`public getCount(): number { return this.items.length; }` - метод считает количество товаров, добавленных в корзину. 
`public hasItem(id: string): boolean { return this.items.some(item => item.id === id); }` - метод проверяет наличие товара в корзине по его id. 

###### Класс Buyer
Модель для работы с данными покупателя. Она хранит сами данные покупателя: вид оплаты, адрес, телефон, эл. почту. А также умеет работать с этими данными. Профиль покупателя соответствет интерфейсу IBuyer.

Поля класса:
` private payment: IBuyer['payment'] = '';` - хранит данные о выбранном способе оплаты;
` private address: string = '';` - хранит адрес;
` private email: string = '';` - хранит эл. почту;
` private phone: string = '';` - хранит номер телефона;

Методы класса:
`public saveData(data: Partial<IBuyer>): void { if (data.payment !== undefined) { this.payment = data.payment; }` - метод хранит данные о покупателе - полные или частичные без удаления других полей. Принимает объект с одним или несколькими полями и обновляет только переданные. 
`public getData(): IBuyer { return { payment: this.payment, address: this.address, email: this.email, phone: this.phone, }; }` - возврашает все поля с данными о покупателе в виде объекта, строго соответствующего интерфейсу IBuyer.
`public clear(): void { this.payment = ''; this.address = ''; this.email = ''; this.phone = ''; }` - полностью сбрасывает данные о покупателе и приводит модель в состояние пустой формы. 
`public validate(): BuyerErrors` - проверяет заполнение каждого отдельного поля с данными от покупателя и возвращает объект с ошибками, если они есть. Возврашает объект, в котором будут только найденные ошибки. Если ошибок нет, вернется пустой объект. 

###### Класс ApiService
Класс для работы с сервером. Использует базовый класс Api через композицию: экземпляр Api передаётся в конструктор и сохраняется в приватном поле. Содержит готовые методы для получения каталога товаров и для отправки заказов. 

Поля класса:
`private api: IApi;` - хранит объект для работы с запросами к серверу, соответствующий интерфейсу IApi;
`constructor(api: IApi) { this.api = api; }` - конструктор принимает готовый объект для запросов и сохраняет его внутри класса для дальнейшей работы с данными. 

Методы класса:
`getProducts(): Promise<IProductsResponse> { return this.api.get<IProductsResponse>("/product/"); }` - отправляет запрос на сервер и возвращает каталог товаров для дальнейшей работы приложения.
`sendOrder(data: IOrderRequest): Promise<IOrderResponse> { return this.api.post<IOrderResponse>("/order/", data, "POST"); }` - отправляет на сервер данные о заказе и возвращает ответ с номером и суммой заказа.
